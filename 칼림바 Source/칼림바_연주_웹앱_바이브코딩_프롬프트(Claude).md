# 🎵 칼림바 연주 웹앱 — 바이브코딩 프롬프트

> 이 문서는 **AI 코딩 도구(Claude, Cursor, v0, Windsurf 등)** 에 그대로 붙여넣어
> "칼림바 연주 웹앱"을 만들기 위한 **완성형 지시서**입니다.
> 아래 1~15번을 순서대로 구현하면 됩니다. 빠르게 시작하려면 맨 끝 **[부록 A]** 의 압축 프롬프트만 복사해도 됩니다.

## 0. 이 문서 사용법

1. 이 마크다운 전체를 복사해서 AI 코딩 도구에 붙여넣습니다.
2. "위 명세대로 단일 HTML 파일로 완성된 칼림바 웹앱을 만들어줘" 라고 요청합니다.
3. 결과물이 나오면 **13번(단계별 개발 순서)** 과 **14번(체크리스트)** 기준으로 검수/보완을 요청합니다.

## 1. 프로젝트 개요

브라우저에서 **키보드(또는 마우스 클릭)로 칼림바(엄지 피아노, Thumb Piano)를 연주**하는 웹앱을 만든다.

- **화면 1 — 선택 화면**: `17키 칼림바` / `21키 칼림바` 중 하나를 고른다.
- **화면 2 — 연주 화면**: 선택한 칼림바가 화면에 그려지고, 키보드 숫자키로 음을 연주한다.
- 핵심 입력 규칙:
  - **숫자키 단독** → 가운데 음역(도~시) 연주
  - **Shift + 숫자키** → 높은 음(C5 이상)
  - **Ctrl + 숫자키** → 낮은 음(B3 이하, 21키 전용)
- 사운드는 **Web Audio API로 합성**(외부 음원 파일 불필요, 오프라인 동작).
- 디자인 톤: **귀엽고 따뜻한(키치/카와이) 분위기**, 둥근 모서리, 부드러운 색감.

## 2. 기술 스택 & 결과물 형태

- **권장: 단일 `index.html` 파일** (HTML + CSS + Vanilla JS, 빌드 과정 없음).
  - 더블클릭만으로 바로 실행되고 배포가 쉬움.
- 사운드: **Web Audio API** (`AudioContext`, `OscillatorNode`, `GainNode`, `BiquadFilterNode`).
- 칼림바 그래픽: **인라인 SVG 또는 HTML/CSS**로 각 건반(tine)을 개별 요소로 렌더링(클릭·애니메이션 가능해야 함).
- 외부 라이브러리/CDN 없이 구현. (원하면 React + Vite로 대체 가능하나, 본 명세는 단일 파일 기준)
- 모바일/데스크톱 반응형. **세로(모바일) 화면을 기본 레퍼런스**로 디자인하되 데스크톱에서도 보기 좋게.

## 3. 화면 흐름


## 4. 공통 디자인 시스템

업로드된 구상 영상의 톤을 따른다. **귀엽고 둥글둥글한 느낌**이 핵심.

**색상 팔레트 (CSS 변수로 정의)**

| 용도 | 색상 | 비고 |
|---|---|---|
| 배경 | `#FBF6EE` (크림/베이지) | 따뜻한 종이 느낌 |
| 포인트(버튼) | `#6FAE4A` ~ `#5C9A3C` (초록) | 선택 라벨·강조 |
| 칼림바 우드 | `#5A4034` ~ `#6E4D3E` (브라운) | 캐릭터 몸체 |
| 금속 건반 | `#D8D8DC` → `#A9A9B0` (실버 그라데이션) | tine |
| 텍스트 | `#3A2E28` (다크 브라운) | |
| 강조/하이라이트 | `#FFD66B` (노란빛 글로우) | 연주 시 빛나는 효과 |

**타이포**: 둥근 산세리프(예: `Pretendard`, `Nunito`, `Quicksand` 계열 또는 시스템 폰트 `system-ui`). 굵고 둥근 느낌.

**공통 스타일**: 큰 라운드 처리(`border-radius: 16~24px`), 부드러운 그림자, 클릭 시 살짝 튀는(bounce) 마이크로 인터랙션.

## 5. [화면 1] 칼림바 선택 화면 상세

**레이아웃**
- 상단: 앱 타이틀(예: "🎵 칼림바 연주하기") + 짧은 안내문("연주할 칼림바를 골라주세요").
- 본문: **두 개의 선택 카드**를 세로(모바일)/가로(데스크톱)로 배치.
  - 카드 A: **17키 칼림바** — 귀여운 칼림바 캐릭터 일러스트(얼굴/볼터치/팔다리) + 초록색 라운드 버튼 라벨 "17키 칼림바".
  - 카드 B: **21키 칼림바** — 동일 스타일, 라벨 "21키 칼림바".
- 캐릭터는 SVG로 간단히 그리거나, 이모지/도형 조합으로 표현해도 됨(브라운 몸체 + 위쪽 금속 건반들 + 웃는 얼굴).

**상호작용**
- 카드 **hover**: 살짝 확대(`scale(1.04)`) + 위로 떠오르는 그림자.
- 카드 **클릭**:
  1. 선택된 카드는 강조(확대 + 완전 채도), 반대 카드는 흐리게(`opacity: 0.4`, 채도↓).
  2. 클릭 지점에 **물결(ripple) 효과**.
  3. 약 0.3~0.5초 후 **화면 2(연주 화면)로 전환**, 선택한 종류(17/21)를 상태로 전달.
- 키보드 접근성: 카드에 `tabindex`, Enter/Space로도 선택 가능.

## 6. [화면 2] 연주 화면 상세

**레이아웃**
- 상단 좌측: **"← 칼림바 다시 선택"** 버튼.
- 중앙: **칼림바 본체**(선택에 따라 17키/21키).
- 하단(또는 토글): **키 안내 패널** — "숫자키 = 가운데 음 / Shift+숫자 = 높은음 / Ctrl+숫자 = 낮은음(21키)" 요약 + 매핑 표.

**칼림바 본체 렌더링 규칙 (중요)**
- 8번의 **음 배치 배열(좌→우 순서)** 대로 건반(tine)을 가로로 나란히 배치한다.
- 각 tine은 **세로 막대** 형태. **음이 낮을수록(가운데일수록) 막대를 길게**, 높을수록 짧게 그려서 칼림바 특유의 "산(∧) 모양"이 나오도록 한다.
  - 막대 길이 ∝ 음 높이 반비례. 예: `length = baseMax - (midi - minMidi) * k` (가운데가 가장 긺).
- tine 아래쪽(자유단)에 **라벨 2줄** 표기:
  - 1줄: 음이름 (예: `C4`, `F3`, `C5`)
  - 2줄: 대응 키 힌트 (예: `1`, `⌃4`, `⇧1`) — Shift는 `⇧`, Ctrl은 `⌃` 기호로.
- tine은 **개별 클릭/터치 가능**해야 하며, 클릭 시 키보드와 동일하게 소리+애니메이션 재생.
- 21키의 **가운데 저음 4개(F3·G3·A3·B3)** 가 가장 긴 막대가 되도록(= 17키 대비 중앙에 더 긴 tine 추가) 그린다.

## 7. 키보드 입력 규칙 (핵심 ★)

세 가지 음역을 **수식키(modifier)** 로 구분한다.

| 입력 | 음역 | 설명 |
|---|---|---|
| `숫자키` 단독 | 가운데 옥타브 (C4~B4) | 도·레·미·파·솔·라·시 |
| `Shift + 숫자키` | 높은 음 (C5 이상) | 한 옥타브 위 + 상단 추가음 |
| `Ctrl + 숫자키` | 낮은 음 (B3 이하) | **21키 전용** 저음 |

**설계 원칙 — "같은 숫자키 = 같은 계이름(도레미)"**
> 숫자 1~7은 항상 **계이름**을 의미한다. (1=도, 2=레, 3=미, 4=파, 5=솔, 6=라, 7=시)
> 수식키가 **옥타브**를 결정한다: 단독=가운데, Shift=위, Ctrl=아래.
> 예) **4번 키 = 항상 '파'** → `Ctrl+4 = F3`, `4 = F4`, `Shift+4 = F5`.
> 가장 높은 음 3개(C6·D6·E6)는 숫자키가 부족하므로 `Shift+8/9/0`에 배정한다.

이렇게 하면 사용자가 규칙을 직관적으로 외울 수 있고, 실제 칼림바 악보(숫자+점 표기)와도 잘 맞는다.

## 8. 음 배치 데이터 (17키 / 21키)

> **표준 C장조 튜닝.** 가운데가 가장 낮은 음(가장 긴 tine), 좌우로 갈수록 높아짐.

**17키 — 화면상 좌→우 순서 (총 17개, 음역 C4~E6)**
```js
const LAYOUT_17 = [
  "D6","B5","G5","E5","C5","A4","F4","D4",
  "C4",                       // ← 가운데(가장 낮음)
  "E4","G4","B4","D5","F5","A5","C6","E6"
];
```

**21키 — 화면상 좌→우 순서 (총 21개, 음역 F3~E6)**
> 17키에 **저음 4개(F3·G3·A3·B3)** 를 가운데에 추가한 구조.
```js
const LAYOUT_21 = [
  "D6","B5","G5","E5","C5","A4","F4","D4",
  "B3","G3","F3","A3",        // ← 가운데 4개(추가된 저음, F3가 가장 낮음)
  "C4","E4","G4","B4","D5","F5","A5","C6","E6"
];
```

**음이름 → MIDI 번호 (주파수 계산용)**
```js
const NOTE_MIDI = {
  F3:53, G3:55, A3:57, B3:59,
  C4:60, D4:62, E4:64, F4:65, G4:67, A4:69, B4:71,
  C5:72, D5:74, E5:76, F5:77, G5:79, A5:81, B5:83,
  C6:84, D6:86, E6:88
};
// 주파수(Hz) = 440 * 2^((midi - 69) / 12)   // A4=69=440Hz 기준
```

## 10. 사운드(오디오) 구현

외부 음원 없이 **Web Audio API 합성**으로 칼림바 음색을 만든다.

**음색 특징(목표)**: 밝고 맑은 금속성 어택 + 부드러운 감쇠. 낮은 음일수록 길게 울림. 동시에 여러 음 가능(폴리포니).

**핵심 요구사항**
- `AudioContext`는 **단 한 번** 생성해 재사용.
- 브라우저 자동재생 정책 때문에 **첫 사용자 입력(키/클릭) 시 `ctx.resume()`** 호출.
- 음 하나당: 기본음 + 배음 몇 개(2배, 3.01배=약간 어긋난 금속 배음, 5.4배) 를 더하고, **빠른 어택 + 지수 감쇠** 엔벨로프 적용. 로우패스 필터로 고역을 살짝 둥글게.
- 새 음을 칠 때 기존 음을 끊지 말 것(자연스럽게 감쇠하도록).

**레퍼런스 구현(JS)**
```js
const ctx = new (window.AudioContext || window.webkitAudioContext)();
const midiToFreq = m => 440 * Math.pow(2, (m - 69) / 12);

function playNote(noteName){
  const midi = NOTE_MIDI[noteName];
  if (midi == null) return;
  if (ctx.state === "suspended") ctx.resume();   // ★ 사용자 제스처 시 깨우기

  const now = ctx.currentTime;
  const f = midiToFreq(midi);
  const dur = Math.max(1.4, 2.6 - (midi - 53) * 0.025); // 낮을수록 길게(약 1.4~2.6s)

  const lp = ctx.createBiquadFilter();
  lp.type = "lowpass";
  lp.frequency.value = Math.min(7000, f * 8);

  const master = ctx.createGain();
  master.connect(lp).connect(ctx.destination);
  master.gain.setValueAtTime(0.0001, now);
  master.gain.exponentialRampToValueAtTime(0.85, now + 0.005); // 빠른 어택
  master.gain.exponentialRampToValueAtTime(0.0001, now + dur); // 지수 감쇠

  const partials = [
    { mul: 1.0,  gain: 1.00 },
    { mul: 2.0,  gain: 0.45 },
    { mul: 3.01, gain: 0.22 },  // 살짝 어긋난 배음 → 금속/타인 느낌
    { mul: 5.4,  gain: 0.08 },  // 고역 반짝임
  ];
  partials.forEach(p => {
    const osc = ctx.createOscillator();
    osc.type = "sine";
    osc.frequency.value = f * p.mul;
    const g = ctx.createGain();
    g.gain.value = p.gain;
    osc.connect(g).connect(master);
    osc.start(now);
    osc.stop(now + dur + 0.1);
  });
}
```
> (선택) 더 사실적인 소리를 원하면 위 합성 대신 음별 샘플(.mp3/.wav)을 로드해 재생하는 방식으로 교체할 수 있음. 기본은 합성 방식.

## 11. 상호작용 & 시각 효과

**연주 시 tine 피드백 (키보드/클릭 공통)**
- 해당 tine을 **살짝 아래로 눌림**(`transform: translateY(3px)`) + **노란빛 글로우**(`box-shadow`/`filter`).
- 끝부분에 **반짝임/물결** 짧게 표시.
- 빠른 **떨림(wobble) 애니메이션**(0.15~0.3s)으로 "퉁겨진" 느낌.
- 키를 떼면 원상복귀.
- (선택) 현재 연주한 음이름을 화면 어딘가에 잠깐 크게 표시.

**키 입력 처리 동작**
- `keydown`에서 소리+애니메이션 트리거, `keyup`에서 시각효과 해제.
- **`event.repeat === true`(꾹 누름)** 는 무시해 같은 음이 연타되지 않게.
- 매핑에 없는 키는 무시. 매핑에 있으면 **`event.preventDefault()`** 로 브라우저 단축키 차단.

**레퍼런스 구현(JS)**
```js
function triggerTine(noteName){
  playNote(noteName);
  const el = document.querySelector(`[data-note="${noteName}"]`);
  if (el){
    el.classList.add("playing");
    setTimeout(() => el.classList.remove("playing"), 220);
  }
}

window.addEventListener("keydown", (e) => {
  const code = e.code;                 // ★ e.key 아님 (Shift+1 → "!")
  const isDigit = /^Digit[0-9]$/.test(code);
  if (!isDigit) return;
  e.preventDefault();                  // 탭전환/저장 등 단축키 차단
  if (e.repeat) return;

  let note = null;
  if (e.ctrlKey){ if (currentKalimba === 21) note = LOW_MAP[code]; }
  else if (e.shiftKey){ note = HIGH_MAP[code]; }
  else { note = MID_MAP[code]; }

  if (note) triggerTine(note);
});
```

## 12. 반드시 지켜야 할 기술적 주의사항 ★

1. **`event.code`를 사용**할 것 (`event.key` 금지).
   - `Shift + 1`을 누르면 `event.key`는 `"!"`로 바뀌지만 `event.code`는 항상 `"Digit1"` 이다. 따라서 물리 키는 `code`, 옥타브 판정은 `shiftKey`/`ctrlKey`로.
2. **브라우저 단축키 충돌 차단**: `Ctrl+숫자`는 기본적으로 브라우저 탭 전환에 쓰인다. 매핑된 키에서 `event.preventDefault()`를 호출해 막는다. (일부 OS/브라우저는 `Ctrl+W` 등 일부 조합을 막지 못할 수 있음 — 주요 타깃인 `Ctrl+4~7`은 대부분 차단 가능.)
3. **AudioContext 자동재생 정책**: 페이지 로드 직후엔 소리가 안 날 수 있다. **첫 키 입력/클릭 때 `ctx.resume()`** 호출(또는 연주 화면 진입 시 사용자 제스처로 초기화).
4. **`event.repeat` 무시**로 키 홀드 시 연타 방지.
5. **macOS 대응(선택)**: Mac 사용자는 `Ctrl` 대신 손이 가는 키가 다를 수 있어, 필요하면 `metaKey`(⌘)도 함께 허용하는 옵션을 둘 수 있음. 기본 명세는 `ctrlKey`.
6. **17키일 때 Ctrl 조합 무시**: 저음이 없으므로 동작하지 않게 한다.
7. **폴리포니/누수 방지**: 각 음의 `OscillatorNode`는 `stop()` 시점을 지정해 자동 정리(메모리 누수 방지).
8. **포커스**: 연주 화면에서 키 입력이 잘 잡히도록 본문이 포커스를 갖게 한다(필요 시 컨테이너에 `tabindex="0"` 후 포커스).

## 13. 단계별 개발 순서 (Phase 1~6)

> 각 단계가 끝날 때마다 동작을 확인하고 다음으로 넘어간다.

**Phase 1 — 뼈대 & 화면 전환**
- 단일 `index.html`에 `#screen-select`, `#screen-play` 두 섹션 생성.
- 화면 1의 두 카드 클릭 → 선택값(`currentKalimba = 17 | 21`) 저장 → 화면 2 표시, "← 다시 선택" 버튼으로 복귀.

**Phase 2 — 데이터 & 칼림바 렌더링**
- `LAYOUT_17`, `LAYOUT_21`, `NOTE_MIDI`, 매핑 객체 정의.
- 선택값에 따라 해당 배열로 tine을 좌→우로 동적 생성. 음 높이에 따라 막대 길이 차등(가운데가 가장 긺). 각 tine에 `data-note` 속성, 라벨 2줄(음이름 + 키힌트) 표시.

**Phase 3 — 사운드 엔진**
- `AudioContext` 1회 생성, `playNote(noteName)` 구현(10번 레퍼런스).
- 첫 사용자 제스처에서 `ctx.resume()`.

**Phase 4 — 키보드 입력**
- `keydown`/`keyup` 핸들러(11·12번 규칙). `code`+`shiftKey`/`ctrlKey`로 음 결정 → `triggerTine`.
- 17키에서 Ctrl 무시, `event.repeat` 무시, `preventDefault`.

**Phase 5 — 마우스/터치 & 시각효과**
- tine 클릭/터치 시 `triggerTine` 호출.
- 눌림/글로우/떨림/물결 애니메이션, keyup 해제.

**Phase 6 — 디자인 마감**
- 4번 팔레트·둥근 스타일 적용, 귀여운 캐릭터 일러스트, 반응형(모바일 세로 우선), 키 안내 패널, 마이크로 인터랙션(bounce/ripple).

## 14. 완성 기준 체크리스트

- [ ] 화면 1에서 17키/21키 선택 → 화면 2로 전환, "다시 선택"으로 복귀된다.

- [ ] 가운데 tine이 가장 길고(낮은 음), 바깥으로 갈수록 짧다(높은 음).
- [ ] 각 tine에 음이름 + 키 힌트(`1`, `⇧1`, `⌃4`)가 표시된다.
- [ ] `1~7` 단독 → C4~B4 소리.
- [ ] `Shift+1~0` → C5~E6 소리.
- [ ] `Ctrl+4~7` → F3~B3 소리(21키에서만), 17키에선 무시.
- [ ] `같은 숫자키 = 같은 계이름` 규칙대로 동작(`Ctrl+4=F3, 4=F4, Shift+4=F5`).
- [ ] tine 클릭/터치로도 같은 음이 난다.
- [ ] 연주 시 눌림/빛남/떨림 애니메이션이 보인다.
- [ ] 여러 키를 빠르게 눌러도 음이 겹쳐 자연스럽게 울린다(폴리포니).
- [ ] 키를 꾹 눌러도 연타되지 않는다.
- [ ] `Ctrl+숫자`로 브라우저 탭이 바뀌지 않는다.
- [ ] 페이지 로드 후 첫 입력에서 소리가 정상적으로 난다(AudioContext resume).
- [ ] 귀엽고 따뜻한 톤의 반응형 UI다.

```
키보드로 연주하는 "칼림바(엄지 피아노) 웹앱"을 외부 라이브러리 없이 단일 index.html(HTML+CSS+Vanilla JS)로 만들어줘. 사운드는 Web Audio API 합성으로.

[화면 흐름]
- 화면1(선택): "17키 칼림바", "21키 칼림바" 두 카드 중 선택. 귀엽고 따뜻한 카와이 톤, 둥근 모서리, 초록 라벨 버튼, 클릭 시 강조+물결 효과 후 화면2로.
- 화면2(연주): 선택한 칼림바를 SVG/CSS로 그림. "← 다시 선택" 버튼으로 화면1 복귀.

[칼림바 그리기]
- 아래 좌→우 순서대로 tine(세로 막대)을 배치. 음이 낮을수록 막대를 길게(가운데가 가장 긺), 높을수록 짧게.
- 17키(C4~E6): D6 B5 G5 E5 C5 A4 F4 D4 [C4=가운데] E4 G4 B4 D5 F5 A5 C6 E6
- 21키(F3~E6): D6 B5 G5 E5 C5 A4 F4 D4 [B3 G3 F3 A3=가운데 저음4] C4 E4 G4 B4 D5 F5 A5 C6 E6
- 각 tine에 음이름과 키 힌트(예: 1, ⇧1, ⌃4) 2줄 표시. tine은 클릭/터치로도 연주 가능.

[키 입력 규칙 — 핵심]
- 숫자 1~7은 항상 계이름(1=도 C, 2=레 D, 3=미 E, 4=파 F, 5=솔 G, 6=라 A, 7=시 B). 수식키가 옥타브 결정.
- 단독: 1~7 = C4~B4
- Shift: Shift+1~7 = C5~B5, Shift+8=C6, Shift+9=D6, Shift+0=E6
- Ctrl(21키 전용): Ctrl+4=F3, Ctrl+5=G3, Ctrl+6=A3, Ctrl+7=B3 (17키에선 Ctrl 무시)
- 같은 숫자키=같은 계이름 유지(Ctrl+4=F3, 4=F4, Shift+4=F5).

[기술 주의]
- 키 감지는 event.key가 아니라 event.code(Digit1~Digit0) + event.shiftKey/event.ctrlKey로. (Shift+1은 key가 "!"라서)
- 매핑된 키에서 event.preventDefault()로 브라우저 단축키(Ctrl+숫자 탭전환 등) 차단.
- event.repeat 무시(꾹 눌러도 연타 금지).
- AudioContext는 1회 생성·재사용, 첫 사용자 입력 때 ctx.resume(). 여러 음 동시 발음(폴리포니), 새 음이 기존 음 끊지 않게.
- 음 주파수 = 440 * 2^((midi-69)/12). MIDI: F3=53,G3=55,A3=57,B3=59,C4=60,D4=62,E4=64,F4=65,G4=67,A4=69,B4=71,C5=72,D5=74,E5=76,F5=77,G5=79,A5=81,B5=83,C6=84,D6=86,E6=88.

[음색]
- 기본음 + 배음(2x, 3.01x, 5.4x), 빠른 어택 + 지수 감쇠(낮은 음일수록 길게, 약 1.4~2.6초), 로우패스로 고역 둥글게 → 맑은 금속성 칼림바 느낌.

[시각효과]
- 연주 시 해당 tine 살짝 눌림 + 노란 글로우 + 짧은 떨림/물결, keyup에 해제. 키보드와 클릭 모두 동일하게 반응.

반응형(모바일 세로 우선), 색감: 크림 배경(#FBF6EE), 초록 포인트(#6FAE4A), 브라운 우드(#5A4034), 실버 건반, 노란 하이라이트(#FFD66B). 둥글둥글 귀여운 톤으로.
```
