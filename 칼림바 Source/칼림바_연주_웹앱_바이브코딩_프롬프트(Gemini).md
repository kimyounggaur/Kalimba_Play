# Role
당신은 UI/UX 디자인 감각이 뛰어나고, Web Audio API에 능숙한 시니어 프론트엔드 개발자입니다. 

# Objective
사용자가 17키 또는 21키 칼림바를 선택하고, PC 키보드와 마우스/터치를 이용해 직접 연주할 수 있는 인터랙티브 웹 애플리케이션을 개발합니다.

# Tech Stack
- Framework: React (또는 Next.js)
- Styling: Tailwind CSS
- Audio: Tone.js 또는 Web Audio API (초기 프로토타입은 샘플러 대신 고품질 Synth로 칼림바와 유사한 맑은 소리를 구현하고, 추후 mp3 매핑이 쉽도록 구조화할 것)

# UI/UX Design System (Core Vibe)
- 전체적인 디자인은 Notion 앱처럼 극도로 깔끔하고 미니멀해야 합니다.
- 배경색은 Off-white(#F9F9F9)를 사용하고, 불필요한 테두리나 그림자를 배제합니다.
- 모바일 세로 화면 비율에 최적화된 레이아웃을 기본으로 합니다.
- 페이지 전환 시 부드러운 Fade-in/out 트랜지션을 적용합니다.

# Page Structure & Features

## 1. Page 1: Kalimba Selection Screen (선택 화면)
- **헤더:** 화면 상단에 심플한 검색창 형태의 장식용 UI와 "Lesson Designer" 텍스트 (영상 프로토타입 참고).
- **메인 레이아웃:** 수직으로 2개의 큰 버튼(또는 카드) 배치.
- **버튼 1 (17키 칼림바):** 귀여운 17키 칼림바 캐릭터 이미지(또는 아이콘)와 하단에 녹색 배경의 "17키 칼림바" 텍스트 라벨.
- **버튼 2 (21키 칼림바):** 귀여운 21키 칼림바 캐릭터 이미지와 하단에 녹색 배경의 "21키 칼림바" 텍스트 라벨.
- **인터랙션:** 마우스 호버 시 살짝 떠오르는 애니메이션, 클릭 시 Page 2로 라우팅 (선택된 칼림바의 종류 상태값 전달).

## 2. Page 2: Play Screen (연주 화면)
- **메인 비주얼:** 화면 중앙에 선택된 칼림바(17키 또는 21키)의 실사풍 이미지 배치.
- **건반(Tine) UI:** 실제 칼림바 쇠막대 위치에 맞게 클릭/터치가 가능한 투명 또는 반투명 오버레이 버튼을 배열합니다.
  - 17키: 중앙(C4)을 기준으로 좌우 번갈아 높아지는 V자 형태 배열.
  - 21키: 17키 배열의 중앙 하단에 F3, G3, A3, B3 건반이 추가된 형태.
- **시각적 피드백:** 키보드나 마우스로 건반을 누를 때, 해당 건반에서 가벼운 파문(Ripple) 이펙트나 색상 강조가 일어나야 합니다.
- **뒤로가기:** 좌측 상단에 미니멀한 '< 뒤로가기' 버튼을 배치하여 선택 화면으로 돌아갈 수 있게 합니다.

# Core Logic: Keyboard Mapping Rule
가장 중요한 핵심 로직입니다. 사용자의 키보드 입력에 따라 정확한 음정이 연주되어야 합니다. 숫자는 다장조(C Major) 기준의 음계(1=도, 2=레...)를 의미합니다.

**[공통 규칙]**
- **기본 숫자키 (1~7):** C4 ~ B4 (1옥타브 도~시)
- **Shift + 숫자키:** C5 이상의 높은 음
- **Ctrl + 숫자키:** B3 이하의 낮은 음

**[상세 매핑 테이블]**
이 매핑 객체를 코드 내에 반드시 명시적으로 구현하세요.

1. **B3 이하 (Ctrl + 숫자) - 21키 전용:**
   - `Ctrl + 4`: F3 (파)
   - `Ctrl + 5`: G3 (솔)
   - `Ctrl + 6`: A3 (라)
   - `Ctrl + 7`: B3 (시)

2. **C4 ~ B4 (기본 숫자 1~7):**
   - `1`: C4 (도)
   - `2`: D4 (레)
   - `3`: E4 (미)
   - `4`: F4 (파)
   - `5`: G4 (솔)
   - `6`: A4 (라)
   - `7`: B4 (시)

3. **C5 이상 (Shift + 숫자):**
   - `Shift + 1`: C5 (높은 도)
   - `Shift + 2`: D5 (높은 레)
   - `Shift + 3`: E5 (높은 미)
   - `Shift + 4`: F5 (높은 파)
   - `Shift + 5`: G5 (높은 솔)
   - `Shift + 6`: A5 (높은 라)
   - `Shift + 7`: B5 (높은 시)
   - `Shift + 8`: C6 (더 높은 도)
   - `Shift + 9`: D6 (더 높은 레)
   - `Shift + 0`: E6 (더 높은 미)

1. 프로젝트의 기본 폴더 구조와 상태 관리(17키 vs 21키 선택) 코드를 작성하세요.
2. 미니멀한 UI 컴포넌트(선택 화면, 연주 화면 레이아웃)를 Tailwind로 마크업 하세요. 이미지 에셋은 우선 임시 플래스홀더(`div` 박스)로 두되 교체가 쉽도록 분리하세요.
3. Web Audio API (또는 Tone.js)를 설정하여 칼림바와 유사한 Synth 소리를 매핑하세요.
4. 명시된 Keyboard Mapping Rule에 따라 전역 `keydown` 이벤트 리스너를 구현하세요. `Shift`와 `Ctrl` 수식키 처리에 유의하세요.
5. 작동이 확인되는 단일 파일 또는 핵심 파일 단위로 코드를 제공해 주세요.
